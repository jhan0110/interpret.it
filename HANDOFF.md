# Phase 7 — Learner-facing surface, Groq cutover, prosody simplification

State as of **2026-05-21**. Live and demonstrable end-to-end against real Claude + Groq + ElevenLabs APIs. Big-picture shift: the product now has a real learner-centric flow (login → home → train → vocab review) and the analysis pipeline collapsed from "two parallel workers" to "one semantic worker derives prosody from ASR word timestamps".

## What shipped since 2026-05-17

### Learner login + modular home hub (frontend)
- **`/login`** (`frontend/app/login/page.tsx`) — UUID input, validates existence via `GET /learners/{id}`, persists to `localStorage['interpretit:learner_id']`, redirects to home. Dev-learner prefill button.
- **`/learner/[learnerId]`** (`frontend/app/learner/[learnerId]/`) — server-rendered hub. Sections:
  - Greeting
  - Feature grid (hero) driven by `features.ts` registry — adding a new feature is one entry in that array
  - Overview: streak tile + minutes-interpreted tile + per-domain mastery bars
  - Recent sessions list
- **`/learner/[learnerId]/train`** — relocated session-creation form. Pre-fills learnerId from URL.
- **Root `/`** is now a client-side redirector to `/login` or `/learner/{id}` based on localStorage.
- **Back-to-home arrows** added on the train + vocab pages via shared `BackToHome.tsx`.

### Vocabulary deck with spaced repetition (full vertical slice)
- **DB (`0003_vocab_deck.py`)** — 3 new tables: `vocab_entries`, `learner_topics`, `learner_vocab_deck`. SM-2 algorithm fields (interval_days, ease_factor, repetitions, next_review_at).
- **Backend** — `services/gateway/app/api/vocab.py`, `app/vocab/seeds.py` (~30 curated terms × 6 domains), `app/vocab/seeder.py`, `app/vocab/srs.py` (SM-2 update).
- **Two SRS input signals**:
  1. Topic seeding — POST `/learners/{id}/vocab/topics` adds ~30 terms for that domain to the learner's deck.
  2. Extraction from failed interpretations — semantic worker enqueues `run_vocab_extraction` whenever `overall_score < 0.75` or errors include `lexical_gap`/`omission`. Claude classifies each missed term as `knowledge_gap` vs `memory_gap` and the gateway upserts it into `learner_vocab_deck` (already-present terms get their interval reset, re-surfacing the card).
- **Frontend** — `/vocab/[learnerId]` flashcard UI with flip + 4-grade SM-2 buttons (Again/Hard/Good/Easy). Topic chips to add/remove domains. Stats grid (due now / total / knowledge gaps / memory gaps).

### Overview backend
- **New endpoint** `GET /learners/{id}/overview` in `services/gateway/app/api/sessions.py` returns `{streak_days, total_seconds_interpreted, mastery_scores}`. Streak = consecutive UTC days ending today with ANY activity (attempt OR vocab review).
- **DB migration** `0004_attempt_duration.py` adds `attempts.duration_ms`; `session_socket.py` now persists `AudioSubmission.duration_ms` (previously discarded).

### ASR migration: faster-whisper → Groq
- New file `services/analysis/app/asr/transcribe_groq.py` — HTTP client for Groq's OpenAI-compatible `/audio/transcriptions` endpoint.
- `transcribe.py` is now a dispatcher on `WHISPER_PROVIDER=groq|local`. Default is `groq`.
- Eliminated the Whisper warm-up cold-start. Per-attempt timeout dropped 90s → 30s.
- New env: `GROQ_API_KEY`, `WHISPER_PROVIDER`, `GROQ_WHISPER_MODEL` (default `whisper-large-v3`).

### Prosody deprecation → word-timestamp derivation
- **Deleted** `services/analysis/app/prosody/pipeline.py` and the `arq-prosody` docker-compose service.
- **New** `services/analysis/app/prosody/word_prosody.py::compute_prosody_from_words` — pure-Python; takes the word-timestamp array Groq already returns and computes pause_count, silence_ratio, mean_wpm, filler_count, then runs the existing classifier. No silero-vad, no librosa, no torch.hub call.
- Prosody push now happens inside `run_semantic` right after Groq returns — cognitive-load dot appears in ~2s, restoring the original "fast path" UX without a second worker.
- Filler lexicon + cognitive_load classifier preserved unchanged.

### Dependency cleanup
Dropped from `services/analysis/pyproject.toml`: `faster-whisper`, `silero-vad`, `librosa`, `spacy`. Torch is now installed CPU-only via `--index-url https://download.pytorch.org/whl/cpu` (saves ~3 GB / image of NVIDIA CUDA libs). Kept: `pydub` (TTS disfluency mixing), `torch` (transitively required by `sentence-transformers`), `sentence-transformers` (paraphrase embeddings).

Net image impact: analysis went from ~5 GB → 2.91 GB.

### Dev infrastructure
- **Bind mounts** added in `docker-compose.yml` for `services/gateway/app`, `services/gateway/alembic`, `services/analysis/app`. Python source changes now require only `docker compose restart <service>` — no rebuild.
- **Image consolidation** — `analysis`, `arq-semantic`, `arq-generation` all share `phase5-analysis:latest` (one `build:` declaration, two `image:` references). Saves ~5.8 GB.

### Contract loosening (defensive)
- `AudioSubmission.audio_format` literal → `str` so Safari MP4/AAC recordings validate
- `AnalysisRequest.register` and `Segment.register` literal → `str` after Claude produced `register="operational"` from a generation prompt and blew up Pydantic validation, breaking attempt enqueue silently

### Frontend polish
- `<audio key={audioUrl}>` to force React to remount on src change (fixed "wrong audio playing" race)
- Two-phase progress bar during listen→record: 0–50% (blue, "Listening") tracks audio playback; 50–100% (yellow, "Recording in Ns") counts down post-audio delay. Record button locked until 100%.
- Synthetic 3-stage analyzing progress bar (Transcribing → Checking → Generating feedback). No real backend events yet — purely time-based against the ~12s SLA. Tagged debug logs (`[WS]`, `[audio]`, `[analyzing]`) throughout.

### Backend debug instrumentation
65 timestamped log statements across 9 files. Tag taxonomy: `[semantic.*]`, `[asr.groq.*]`, `[eval.claude.*]`, `[reference.claude.*]`, `[tts.*]`, `[rpc.push_*.*]`, `[gw.internal.*]`, `[gw.close_if_ready.*]`, `[ws.state_change.sent]`, `[prosody.derived]`. Each includes `attempt=<id>` so concurrent attempts can be greppped independently:
```bash

```

## Current architecture quick-glance

```
Browser ──WS── Gateway ──Redis──> arq-semantic
                  │                    │
                  │                    ├── Groq /audio/transcriptions (ASR + words)
                  │                    ├── compute_prosody_from_words() → POST /internal/prosody_result
                  │                    ├── Redis (reference cache, 24h)
                  │                    ├── Claude (reference if miss)
                  │                    ├── ElevenLabs (feedback TTS)
                  │                    ├── Claude (evaluate) → POST /internal/semantic_result
                  │                    └── (conditional) enqueue run_vocab_extraction
                  └──> arq-generation (10-pack pre-gen, Claude + ElevenLabs)
```
